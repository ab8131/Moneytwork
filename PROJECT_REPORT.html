3.1 Market Data Tracking

3.2 Portfolio Management

Users record buy and sell transactions for any tracked asset. Transaction recording supports dual input: users enter either quantity or total amount, with automatic calculation of the complementary value. Portfolio calculates real-time profit/loss based on current market prices. Holdings tracked per asset with average cost basis calculation. Portfolio overview displays total investment, current value, and overall return percentage.

3.3 Authentication System

Firebase Authentication handles user management via email and password. Registration requires username, email, and password (minimum 6 characters). User profiles stored in Firestore with username, email, and creation timestamp. Session persistence maintains login state across app restarts. Profile screen displays user information with logout capability.

3.4 Real-time Chat

Each cryptocurrency and stock has a dedicated chat room. Messages synchronized in real-time using Firestore snapshot listeners. Messages display sender username and timestamp. System maintains last 100 messages per chat room. Non-authenticated users have read-only access; authenticated users can send messages.

Portfolio calculations: Average cost basis = total spent ÷ total quantity. Current value = quantity × current price. Profit/loss = current value - total invested. Percentage return = ((current - invested) ÷ invested) × 100. Each transaction stored with original price, quantity, timestamp.

5.2 Real-time Synchronization

Chat uses Firestore addSnapshotListener for real-time updates. Messages written to Firestore propagate to all connected clients within seconds. Messages ordered by timestamp. Automatic reconnection on network restoration. Limited to 100 most recent messages per room.
